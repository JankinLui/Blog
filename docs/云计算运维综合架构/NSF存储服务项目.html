    NSF存储服务项目

    贪吃偷懒的猫 2023/1/7 云计算运维综合架构LinuxNSF

    # 第1章 NSF软件介绍

    # 1.1 什么是NSF

    NSF是Network File System的缩写，中文意思是网络文件共享系统，它的主要功能是通过网络(一般是局域网)让不同的主机系统之间可以共享文件或目录

    # 1.1.1 储存服务的种类

    名称 解释
    FTP 文件传输协议
    samba windows--Linux之间数据传输
    NFS Linux--Linux之间数据传输

    samba具体部署可以看这篇文章linux部署samba (opens new window)

    # 1.1.2 利用分布式储存

    Moosefs（mfs） 比较落伍， GlusterFS FastDFS 企业应用多

    # 1.1.3 原理示意图

    # 1.1.4 NFS服务工作原理

    服务端：

    1. 启动rpc服务，开启111端口
    2. 启动nfs服务
    3. 实现nfs服务进程和端口好的注册

    补充：检查nfs服务进程与端口注册信息 rpcinfo -p localhost //查看本机的nfs进行及使用端口信息

    客户端：

    1. 与服务端建立TCP网络链接
    2. 客户端执行挂载命令，进行远程挂载
    3. 可以实现数据远程传输储存

    # 1.2 NSF储存服务的作用及参数说明

    1. 实现数据的共享储存
    2. 编写数据操作管理
    3. 节省购买服务器磁盘开销

    # 1.2.1 NFS配置参数权限

    权限参数 含义
    rw 储存目录是否有读写权限
    ro 储存目录是否只读权限
    sync 同步方式存储数据 直接将数据保存到硬盘(数据是安全的)
    async 异步方式存储数据 直接将数据保存到内存(提高数据储存效率)
    no_root_squash 不要将root用户身份进行转换
    root_aquash 将root用户身份进行转换
    all_squash 将所有用户身份都进行转换
    no_all_squash 不要将普通用户身份进行转换
    anonuid 指定的是匿名用户的uid(数字)
    anongid 指定的是匿名用户组的gid（数字）

    # 1.2.2 NFS参数配置

    基本配置

    vim /etc/exports
/data     172.16.1.0/24(rw,sync,no_all_squash) 
修改映射用户配置：www=1002
/data 172.16.1.0/24(rw,sync,anonuid=1002,anongid=1002)

    # 1.2.4 客户端mount命令参数

    参数 含义
    rw 实现挂载后挂载点目录可读可写（默认）
    ro 实现挂载后挂载点目录可读
    suid 在共享目录中可以让setuid权限位生效（默认）
    nosuid 在共享目录中可以让setuid权限位失效,提供共享目录的安全
    exec 共享目录中的执行文件可以直接执行
    noexec 共享目录中的执行文件无法直接执行
    auto 可以实现自动挂载(mount -a 实现自动加载)
    noauto 不可以实现自动挂载
    nouser 禁止普通用户可以卸载挂载点
    user 允许普通用户可以卸载挂载点

    # 2.2.1 保证网站储存服务器用户数据安全性：

    no_all_squash 需要进行配置 共享目录权限为www root_squash 需要进行配置 root---nfsnobody data目录---www(uid要设置一样)

    # 2.2.2 企业中如何编辑nfs配置文件

    1. 通用方法
    /data 172.16.1.0/24(rw,sync)
    1
    1. 特殊情况（让部分人员不能操作储存目录 可以看目录中的数据）
    /data 10.0.0.0/24(ro,sync)
    1
    1. 修改默认的匿名用户
    /data 10.0.0.0/24(ro,sync,anonuid=xxx,anongid=xxx)
    1

    # 第3章 NSF服务问题总结

    # 3.1 共享目录的权限和哪些因素有关

    1. 和储存目录的本身权限有关(755 属主：nfsnobody)
    2. 和配置文件中的权限配置有关(rw/ro xxx_squash anonuid/anongid)

    # 3.2nfs服务器重启，挂载后创建数据比较慢

    服务器重启方式不正确 服务重启：

    重启命令 作用 效果 用户感受
    restart 重启服务 强制断开所有连接 用户感受不好
    reload 重启服务 强制断开没用数据传输的连接 ~

    # 3.3 无法实现网络存储服务自动挂载原因

    # centos6：

    根据系统服务启动顺序（串行） 按照顺序依次启动 network服务 --- sshd服务 --- crond服务 --- rsync服务 --- rpcbind服务 --- nfs服务 先加载/etc/fstab --- network服务 --- autofs服务

    autofs服务程序：开机自启动 服务启动好之后，重新加载fstab

    # centos7：

    根据系统服务启动顺序（并行） neywork服务 sshd服务 crond服务 rsync服务 先加载/etc/fstab network服务

    # 3.4 小结

    # 1、NFS存储服务概念介绍

    a. 实现数据的共享存储 b. 降低公司磁盘购买成本

    # 2、NFS存储服务工作原理

    a. 部署好一台存储服务器，设置好存储目录 b. 客户端利用网络挂载的方式进行挂载储存目录 c. 将数据存储在客户端本地挂载点目录==存储到存储服务器中
