    # 第二章 中小规模网站架构组成

    1. 顾客--用户

      访问网站的人员

    2. 保安--防火墙（firewalld）

      进行访问策略控制

    3. 迎宾--负载均衡服务器（nginx）

      对用户的访问请求进行调度处理

    4. 服务员--web服务器（nginx）

      处理用户的请求

    5. 厨师--数据库服务器（mysql）

      存储字符数据（耳机 500 索尼 黑色 北京地址 订单时间2022-08-04 17:22:20）

    6. 厨师--存储服务器（nfs）

      存储图片 音频 视频 附件等数据信息

    7. 厨师--备份服务器（rsync+crond-定时服务 rsync+sersync-实时备份）

      存储网站所有服务器的重要数据

    8. 厨师--缓存服务器（memcache redis mongodb）

      • 将数据信息存储到内存中
      • 减缓服务器的压力

    9. 经理--批量管理服务器（ansible）

      批量管理多台服务器主机

    9. 模板机优化配置---加大文件描述

      加大文件描述（加大进程的加载，比如nofile默认是打开1024个文件，当程序运行足够多时，到达1024后再打开程序将无法运行）

      echo '*                  -                       nofile                              65535' >>/etc/security/limits.conf
tail -l /etc/security/limits.conf
      1
      2

      说明

      一个服务程序运行起来，会打开相应的文件

      crond定时任务服务 --- systemctl start crond --- 打开相应的文件

      /var/spool/cron/root ---加载打开配置文件

      /var/log/cron ---加载打开日志文件

    # 第十章 编写全网备份脚本

    # 10.1 客户端脚本

    mkdir /server/scripts -p     //建立一个专门放脚本的文件
vim backup.sh
    1
    2
    #!/bin/bash
			
Backup_dir="/backup"                //定义变量，方便更换
IP_info=$(hostname -i)
			
#create backup dir
mkdir -p $Backup_dir/$IP_info/
			
#tar backup data
cd /
tar zcvf $Backup_dir/$IP_info/date_$(date +%F_week%w -d -1day).tar.gz /etc/hosts
			
#del 7 day ago date
find $Backup_dir -type f -mtime +7 | xargs rm 2>/dev/null
			
#create finger file
find $Backup_dir/ -type f -mtime -1 ! -name "finger*" | xargs md5sum > $Backup_dir/$IP_info/finger.txt
			
#backup push data info
rsync -avz $Backup_dir/ rsync_backup@172.16.1.41::backup --password-file=/etc/rsync.password
